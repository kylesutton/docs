```
URL: http://<db.host>:<db.port>/has/type
```
Check for the existence of a type.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| type_id | string | Id of the type returned in response to [/create/type](/content/api/rest/create_type_rest#create-type-rest) request. |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). |


## Output Parameter Description


The GPUdb server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

| Name | Type | Description |
| --- | --- | --- |
| status | String | 'OK' or 'ERROR' |
| message | String | Empty if success or an error message |
| data_type | String | 'has_type_response' or 'none' in case of an error |
| data | String | Empty string |
| data_str | JSON or | This embedded JSON represents the result of the /has/type endpoint: <br/> <br/> type_id: [string] Value of input parameter *type_id*. <br/> <br/> type_exists: [boolean] Indicates whether the type exists or not.  Possible values are: * true * false <br/> <br/> info: [map of string to strings] Additional information. <br/> <br/> Empty string in case of an error. |
