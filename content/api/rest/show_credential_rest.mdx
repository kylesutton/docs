```
URL: http://<db.host>:<db.port>/show/credential
```
Shows information about a specified [credential](../../../concepts/credentials/) or all credentials.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| credential_name | string | Name of the credential on which to retrieve information. The name must refer to a currently existing credential. If '*' is specified, information about all credentials will be returned. |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). |


## Output Parameter Description


The GPUdb server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

| Name | Type | Description |
| --- | --- | --- |
| status | String | 'OK' or 'ERROR' |
| message | String | Empty if success or an error message |
| data_type | String | 'show_credential_response' or 'none' in case of an error |
| data | String | Empty string |
| data_str | JSON or | This embedded JSON represents the result of the /show/credential endpoint: <br/> <br/> credential_names: [array of strings] A list of all credential names. <br/> <br/> credential_types: [array of strings] A list of each credential's type. <br/> <br/> credential_identities: [array of strings] A list of each credential's identity. <br/> <br/> credentials: [array of strings] A list of each credential's create_credential_request JSON encoded structure. <br/> <br/> additional_info: [array of maps of string to strings] Additional information about the respective credential in output parameter *credential_names*. <br/> <br/> info: [map of string to strings] Additional information. <br/> <br/> Empty string in case of an error. |
