```
URL: http://<db.host>:<db.port>/lock/table
```
Manages global access to a table's data.  By default a table has a input parameter *lock_type* of *read_write*, indicating all operations are permitted.  A user may request a *read_only* or a *write_only* lock, after which only read or write operations, respectively, are permitted on the table until the lock is removed.  When input parameter *lock_type* is *no_access* then no operations are permitted on the table.  The lock status can be queried by setting input parameter *lock_type* to *status*.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| table_name | string | Name of the table to be locked, in [schema_name.]table_name format, using standard [name resolution rules](../../../concepts/tables/#table-name-resolution).  It must be a currently existing table or view. |
| lock_type | string | The type of lock being applied to the table. Setting it to *status* will return the current lock status of the table without changing it. The default value is `status`. <br/> <br/> status: Show locked status <br/> no_access: Allow no read/write operations <br/> read_only: Allow only read operations <br/> write_only: Allow only write operations <br/> read_write: Allow all read/write operations |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). |


## Output Parameter Description


The GPUdb server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

| Name | Type | Description |
| --- | --- | --- |
| status | String | 'OK' or 'ERROR' |
| message | String | Empty if success or an error message |
| data_type | String | 'lock_table_response' or 'none' in case of an error |
| data | String | Empty string |
| data_str | JSON or | This embedded JSON represents the result of the /lock/table endpoint: <br/> <br/> lock_type: [string] Returns the lock state of the table. <br/> info: [map of string to strings] Additional information. <br/> Empty string in case of an error. |
