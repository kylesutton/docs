```
URL: http://<db.host>:<db.port>/admin/show/jobs
```
Get a list of the current jobs in GPUdb.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). <br/> <br/> show_async_jobs: If *true*, then the completed async jobs are also included in the response. By default, once the async jobs are completed they are no longer included in the jobs list. The default value is `false`. The supported values are: * true * false <br/> show_worker_info: If *true*, then information is also returned from worker ranks. By default only status from the head rank is returned.  The supported values are: * true * false |


## Output Parameter Description


The GPUdb server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

| Name | Type | Description |
| --- | --- | --- |
| status | String | 'OK' or 'ERROR' |
| message | String | Empty if success or an error message |
| data_type | String | 'admin_show_jobs_response' or 'none' in case of an error |
| data | String | Empty string |
| data_str | JSON or | This embedded JSON represents the result of the /admin/show/jobs endpoint: <br/> <br/> job_id: [array of longs] <br/> status: [array of strings] <br/> endpoint_name: [array of strings] <br/> time_received: [array of longs] <br/> auth_id: [array of strings] <br/> source_ip: [array of strings] <br/> query_text: [array of strings] <br/> user_data: [array of strings] <br/> flags: [array of strings] <br/> info: [map of string to strings] Additional information. The default value is an empty map ( \{\} ). +------------+--------------------------------------------------------------------------------------------------------------------------------------+ \|Possible \|Parameter Description \| \|Parameters \| \| \|(keys) \| \| +============+======================================================================================================================================+ \|job_tag \|The job tag specified by the user or if unspecified by user, an internally generated unique identifier for the job across clusters. \| +------------+--------------------------------------------------------------------------------------------------------------------------------------+ \|worker_info \|Worker job information as json \| +------------+--------------------------------------------------------------------------------------------------------------------------------------+ <br/> Empty string in case of an error. |
