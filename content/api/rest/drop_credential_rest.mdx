```
URL: http://<db.host>:<db.port>/drop/credential
```
Drop an existing [credential](../../../concepts/credentials/).


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| credential_name | string | Name of the credential to be dropped. Must be an existing credential. |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). |


## Output Parameter Description


The GPUdb server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

| Name | Type | Description |
| --- | --- | --- |
| status | String | 'OK' or 'ERROR' |
| message | String | Empty if success or an error message |
| data_type | String | 'drop_credential_response' or 'none' in case of an error |
| data | String | Empty string |
| data_str | JSON or | This embedded JSON represents the result of the /drop/credential endpoint: <br/> <br/> credential_name: [string] Value of input parameter *credential_name*. <br/> info: [map of string to strings] Additional information. <br/> Empty string in case of an error. |
