<a id="grant-permission-rest"></a>


```
URL: http://<db.host>:<db.port>/grant/permission
```
Grant user or role the specified permission on the specified object.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| principal | string | Name of the user or role for which the permission is being granted.  Must be an existing user or role.  The default value is ''. |
| object | string | Name of object permission is being granted to.  It is recommended to use a fully-qualified name when possible. |
| object_type | string | The type of object being granted to <br/> <br/> catalog: Catalog <br/> context: Context <br/> credential: Credential <br/> datasink: Data Sink <br/> datasource: Data Source <br/> directory: KIFS File Directory <br/> graph: A Graph object <br/> proc: UDF Procedure <br/> schema: Schema <br/> sql_proc: SQL Procedure <br/> system: System-level access <br/> table: Database Table <br/> table_monitor: Table monitor |
| permission | string | Permission being granted. <br/> <br/> admin: Full read/write and administrative access on the object. <br/> connect: Connect access on the given data source or data sink. <br/> create: Ability to create new objects of this type. <br/> delete: Delete rows from tables. <br/> execute: Ability to Execute the Procedure object. <br/> insert: Insert access to tables. <br/> monitor: Monitor logs and statistics. <br/> read: Ability to read, list and use the object. <br/> send_alert: Ability to send system alerts. <br/> update: Update access to the table. <br/> user_admin: Access to administer users and roles that do not have system_admin permission. <br/> write: Access to write, change and delete objects. |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). <br/> <br/> columns: Apply table security to these columns, comma-separated.  The default value is ''. <br/> filter_expression: Optional filter expression to apply to this grant.  Only rows that match the filter will be affected.  The default value is ''. <br/> with_grant_option: Allow the recipient to grant the same permission (or subset) to others The default value is `false`. The supported values are: * true * false |


## Output Parameter Description


The GPUdb server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

| Name | Type | Description |
| --- | --- | --- |
| status | String | 'OK' or 'ERROR' |
| message | String | Empty if success or an error message |
| data_type | String | 'grant_permission_response' or 'none' in case of an error |
| data | String | Empty string |
| data_str | JSON or | This embedded JSON represents the result of the /grant/permission endpoint: <br/> <br/> principal: [string] Value of input parameter *principal*. <br/> object: [string] Value of input parameter *object*. <br/> object_type: [string] Value of input parameter *object_type*. <br/> permission: [string] Value of input parameter *permission*. <br/> info: [map of string to strings] Additional information. <br/> Empty string in case of an error. |
