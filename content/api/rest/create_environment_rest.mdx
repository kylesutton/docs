```
URL: http://<db.host>:<db.port>/create/environment
```
Creates a new environment which can be used by [user-defined functions](../../../concepts/udf/) (UDF).


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| environment_name | string | Name of the environment to be created. |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). |


## Output Parameter Description


The GPUdb server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

| Name | Type | Description |
| --- | --- | --- |
| status | String | 'OK' or 'ERROR' |
| message | String | Empty if success or an error message |
| data_type | String | 'create_environment_response' or 'none' in case of an error |
| data | String | Empty string |
| data_str | JSON or | This embedded JSON represents the result of the /create/environment endpoint: <br/> <br/> environment_name: [string] Value of input parameter *environment_name*. <br/> info: [map of string to strings] Additional information. <br/> Empty string in case of an error. |
