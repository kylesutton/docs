```
URL: http://<db.host>:<db.port>/admin/send/alert
```
Sends a user generated alert to the monitoring system.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| message | string | Alert message body.  The default value is ''. |
| label | string | Label to add to alert message.  The default value is ''. |
| log_level | string | Alert message logging criteria  The supported values are: <br/> * fatal <br/> * error <br/> * warn <br/> * info <br/> * debug |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). |


## Output Parameter Description


The GPUdb server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

| Name | Type | Description |
| --- | --- | --- |
| status | String | 'OK' or 'ERROR' |
| message | String | Empty if success or an error message |
| data_type | String | 'admin_send_alert_response' or 'none' in case of an error |
| data | String | Empty string |
| data_str | JSON or String | This embedded JSON represents the result of the /admin/send/alert endpoint: <br/> <br/> info: [map of string to strings] Additional information. <br/> Empty string in case of an error. |
