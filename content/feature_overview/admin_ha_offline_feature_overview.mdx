<a id="admin-ha-offline-feature-overview"></a>


Pauses consumption of messages from other HA clusters to support
data repair/recovery scenarios. In-flight queries may fail to replicate to
other clusters in the ring when going offline.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| offline | boolean | Set to true if desired state is offline.  The supported values are: <br/> <br/> * true <br/> * false |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). |


## Output Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| info | map of string to strings | Additional information. |
