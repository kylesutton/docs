<a id="grant-permission-feature-overview"></a>


Grant user or role the specified permission on the specified object.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| principal | string | Name of the user or role for which the permission is being granted.  Must be an existing user or role.  The default value is ''. |
| object | string | Name of object permission is being granted to.  It is recommended to use a fully-qualified name when possible. |
| object_type | string | The type of object being granted to <br/> <br/> catalog: Catalog <br/> <br/> context: Context <br/> <br/> credential: Credential <br/> <br/> datasink: Data Sink <br/> <br/> datasource: Data Source <br/> <br/> directory: KIFS File Directory <br/> <br/> graph: A Graph object <br/> <br/> proc: UDF Procedure <br/> <br/> schema: Schema <br/> <br/> sql_proc: SQL Procedure <br/> <br/> system: System-level access <br/> <br/> table: Database Table <br/> <br/> table_monitor: Table monitor |
| permission | string | Permission being granted. <br/> <br/> admin: Full read/write and administrative access on the object. <br/> <br/> connect: Connect access on the given data source or data sink. <br/> <br/> create: Ability to create new objects of this type. <br/> <br/> delete: Delete rows from tables. <br/> <br/> execute: Ability to Execute the Procedure object. <br/> <br/> insert: Insert access to tables. <br/> <br/> monitor: Monitor logs and statistics. <br/> <br/> read: Ability to read, list and use the object. <br/> <br/> send_alert: Ability to send system alerts. <br/> <br/> update: Update access to the table. <br/> <br/> user_admin: Access to administer users and roles that do not have system_admin permission. <br/> <br/> write: Access to write, change and delete objects. |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). <br/> <br/> columns: Apply table security to these columns, comma-separated.  The default value is ''. <br/> <br/> filter_expression: Optional filter expression to apply to this grant.  Only rows that match the filter will be affected.  The default value is ''. <br/> <br/> with_grant_option: Allow the recipient to grant the same permission (or subset) to others The default value is `false`. The supported values are: * true * false |


## Output Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| principal | string | Value of input parameter *principal*. |
| object | string | Value of input parameter *object*. |
| object_type | string | Value of input parameter *object_type*. |
| permission | string | Value of input parameter *permission*. |
| info | map of string to strings | Additional information. |
