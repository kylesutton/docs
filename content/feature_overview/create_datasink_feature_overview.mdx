<a id="create-datasink-feature-overview"></a>


Creates a [data sink](../../concepts/data_sinks/), which contains the
destination information for a data sink that is external to the database.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| name | string | Name of the data sink to be created. |
| destination | string | Destination for the output data in format 'storage_provider_type://path[:port]'.  Supported storage provider types are 'azure', 'gcs', 'hdfs', 'http', 'https', 'jdbc', 'kafka', and 's3'. |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). <br/> <br/> connection_timeout: Timeout in seconds for connecting to this data sink <br/> <br/> wait_timeout: Timeout in seconds for waiting for a response from this data sink <br/> <br/> credential: Name of the [credential](../../concepts/credentials/) object to be used in this data sink <br/> <br/> s3_bucket_name: Name of the Amazon S3 bucket to use as the data sink <br/> <br/> s3_region: Name of the Amazon S3 region where the given bucket is located <br/> <br/> s3_verify_ssl: Whether to verify SSL connections The default value is `true`. +-----------+---------------------------------------------------------------------------------------------------------------------------+ \|Supported \|Description \| \|Values \| \| +===========+===========================================================================================================================+ \|true \|Connect with SSL verification \| +-----------+---------------------------------------------------------------------------------------------------------------------------+ \|false \|Connect without verifying the SSL connection; for testing purposes, bypassing TLS errors, self-signed certificates, etc. \| +-----------+---------------------------------------------------------------------------------------------------------------------------+ <br/> <br/> s3_use_virtual_addressing: Whether to use virtual addressing when referencing the Amazon S3 sink The default value is `true`. +-----------+-----------------------------------------------------------------------------------------------------------------------------------+ \|Supported \|Description \| \|Values \| \| +===========+===================================================================================================================================+ \|true \|The requests URI should be specified in virtual-hosted-style format where the bucket name is part of the domain name in the URL. \| +-----------+-----------------------------------------------------------------------------------------------------------------------------------+ \|false \|Use path-style URI for requests. \| +-----------+-----------------------------------------------------------------------------------------------------------------------------------+ <br/> <br/> s3_aws_role_arn: Amazon IAM Role ARN which has required S3 permissions that can be assumed for the given S3 IAM user <br/> <br/> s3_encryption_customer_algorithm: Customer encryption algorithm used encrypting data <br/> <br/> s3_encryption_customer_key: Customer encryption key to encrypt or decrypt data <br/> <br/> s3_encryption_type: Server side encryption type <br/> <br/> s3_kms_key_id: KMS key <br/> <br/> hdfs_kerberos_keytab: Kerberos keytab file location for the given HDFS user.  This may be a KIFS file. <br/> <br/> hdfs_delegation_token: Delegation token for the given HDFS user <br/> <br/> hdfs_use_kerberos: Use kerberos authentication for the given HDFS cluster The default value is `false`.  The supported values are: * true * false <br/> <br/> azure_storage_account_name: Name of the Azure storage account to use as the data sink, this is valid only if tenant_id is specified <br/> <br/> azure_container_name: Name of the Azure storage container to use as the data sink <br/> <br/> azure_tenant_id: Active Directory tenant ID (or directory ID) <br/> <br/> azure_sas_token: Shared access signature token for Azure storage account to use as the data sink <br/> <br/> azure_oauth_token: Oauth token to access given storage container <br/> <br/> gcs_bucket_name: Name of the Google Cloud Storage bucket to use as the data sink <br/> <br/> gcs_project_id: Name of the Google Cloud project to use as the data sink <br/> <br/> gcs_service_account_keys: Google Cloud service account keys to use for authenticating the data sink <br/> <br/> jdbc_driver_jar_path: JDBC driver jar file location <br/> <br/> jdbc_driver_class_name: Name of the JDBC driver class <br/> <br/> kafka_topic_name: Name of the Kafka topic to publish to if input parameter *destination* is a Kafka broker <br/> <br/> max_batch_size: Maximum number of records per notification message.  The default value is '1'. <br/> <br/> max_message_size: Maximum size in bytes of each notification message.  The default value is '1000000'. <br/> <br/> json_format: The desired format of JSON encoded notifications message. The default value is `flat`. +-----------+-----------------------------------------------+ \|Supported \|Description \| \|Values \| \| +===========+===============================================+ \|flat \|A single record is returned per message \| +-----------+-----------------------------------------------+ \|nested \|Records are returned as an array per message \| +-----------+-----------------------------------------------+ <br/> <br/> use_managed_credentials: When no credentials are supplied, we use anonymous access by default. If this is set, we will use cloud provider user settings. The default value is `false`. The supported values are: * true * false <br/> <br/> use_https: Use https to connect to datasink if true, otherwise use http The default value is `true`.  The supported values are: * true * false <br/> <br/> skip_validation: Bypass validation of connection to this data sink. The default value is `false`.  The supported values are: * true * false |


## Output Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| name | string | Value of input parameter *name*. |
| info | map of string to strings | Additional information. |
