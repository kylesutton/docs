title: Revoke Table Permission
---

{/* RST directive: role h2_style */}


<a id="revoke-permission-table-feature-overview"></a>


Revokes a table-level permission from a user or role.


## Input Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| name | string | Name of the user or role from which the permission will be revoked. Must be an existing user or role. |
| permission | string | Permission to revoke from the user or role. <br/> <br/> table_admin: Full read/write and administrative access to the table. <br/> table_insert: Insert access to the table. <br/> table_update: Update access to the table. <br/> table_delete: Delete access to the table. <br/> table_read: Read access to the table. |
| table_name | string | Name of the table to which the permission grants access, in [schema_name.]table_name format, using standard [name resolution rules](../../concepts/tables/#table-name-resolution).  Must be an existing table, view or schema. |
| options | map of string to strings | Optional parameters.  The default value is an empty map ( \{\} ). <br/> <br/> columns: Apply security to these columns, comma-separated.  The default value is ''. |


## Output Parameter Description


| Name | Type | Description |
| --- | --- | --- |
| name | string | Value of input parameter *name*. |
| permission | string | Value of input parameter *permission*. |
| table_name | string | Value of input parameter *table_name*. |
| info | map of string to strings | Additional information. |
